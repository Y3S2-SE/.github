## Repository Guidelines
- Each project must have its own repository
- Include a proper `README.md` per repository
- Use meaningful commit messages
- Follow agreed coding standards
- Do not push sensitive data (keys, passwords, tokens)

## Branching & Workflow
- `main` → stable, reviewed code only
- Feature development should use separate branches
- Merge via pull requests when possible
